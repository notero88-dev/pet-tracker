- ✅ **Traccar auth switched from session cookies to HTTP Basic auth** (`lib/services/traccar_api.dart`). On Flutter web, browsers hide `Set-Cookie` from JavaScript and won't forward cross-origin cookies without `withCredentials=true` + matching CORS on Traccar. Basic auth sidesteps all of that and works identically on iOS/Android/web. The previous cookie-based flow was extracting `Set-Cookie` from the login response and re-sending it as `Cookie` on subsequent requests — this silently returned `null` on web (browser security policy), leading to 401s on every Traccar call. Change: `_sessionCookie` field + cookie extraction replaced with `_authHeader` storing a pre-encoded `Basic <base64(email:password)>` string; `_headers` getter now sends `Authorization` instead of `Cookie`. No API surface change; no changes to any other files. Documented in the class-level comment of `traccar_api.dart`.
